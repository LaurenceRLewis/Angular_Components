Structural Directives & Semantic HTML

Angular provides structural directives like *ngIf and *ngFor for conditional and dynamic rendering. When combined with semantic HTML and invisible containers like <ng-container>, you can build powerful and valid UI structures.

Invisible Containers

  • <ng-container> – no DOM output, great for logic grouping
  • <ng-template> – defines templates rendered later with *ngIf or ngTemplateOutlet
  • <ng-component> – dynamically rendered shell (not manually used)

Valid HTML Examples

❌ Invalid✅ Valid
<table><div *ngIf="x"> <table><ng-container *ngIf="x">
<tr><ng-container></ng-container></tr> <tr *ngIf="row">...</tr>

Content Projection (Slots)

Angular’s <ng-content> is equivalent to Web Component slots. It allows custom, flexible content to be passed into a reusable layout. 


  <section class="card">
    <ng-content select="[header]"></ng-content>
    <ng-content select="[body]"></ng-content>
    <ng-content select="[footer]"></ng-content>
  </section>

This keeps the component clean and adaptable for different consumers.

Maintainability Guidelines

  • Use *ngFor on valid, semantic tags (e.g. <tr>, <li>)
  • Use <ng-container> for logic, not layout
  • Don’t break table semantics with <div> or <span> between rows
  • Use <ng-content> to expose layout flexibility to consumers
