VoiceOver (iOS)

HTML tables involves finger gestures and the VoiceOver rotor. When VoiceOver focus lands on a table, it will announce the table and often its dimensions (e.g. “Table, 4 rows, 3 columns”).
For more controlled navigation, use the Rotor. Rotate two fingers on the screen to choose table-specific navigation options.

Gesture Action
Swipe RightMove to the next cell in the row​
Swipe LeftMove to the previous cell in the row​
Rotor to "Tables", Swipe Down/UpJump to next/previous table on the page
Rotor to "Rows", Swipe DownMove to the cell below in the column​
Rotor to "Rows", Swipe UpMove to the cell above in the column
Explore by touchDrag a finger across the table to hear cell contents (useful for randomly accessing cells by their spatial layout)

TalkBack (Android)

TalkBack, the Android screen reader, lets users swipe through page elements, including tables. When focus moves into a table, TalkBack announces it as a table and usually reports the number of rows and columns (e.g. “Table with 5 rows and 3 columns”).​
It also identifies headers: as you move through cells, TalkBack will prepend the associated header text to that cell's announcement if headers are present​

Gesture Action
Swipe RightMove to next cell (in reading order)​
Swipe LeftPrevious cell (move left to previous cell)​
Swipe Down/Up(No direct vertical navigation by swipe – changes navigation granularity)
Explore by touch (drag finger)Move to a specific cell above or below (manual touch navigation for vertical movement)​

Note: TalkBack's local context menu (Reading Controls) can be opened with an angle gesture (up then right). This menu allows jumping by elements like headings, links, etc., but it does not provide a straightforward "next row" command for tables. The primary method to traverse tables is swiping horizontally through cells​.
